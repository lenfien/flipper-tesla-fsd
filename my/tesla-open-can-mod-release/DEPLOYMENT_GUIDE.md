## 我们的方案有什么不同？

市面上有不少 CAN mod 方案，以下是我们做的改进：

### 1. 车机 + 模块 + 路由器 = 局域网

三个设备组成一个专属局域网。你可以直接在 **车机浏览器** 里打开模块的 Web 控制台，实时切换所有配置——不需要掏手机，不需要连电脑，开车的时候就能改。

### 2. 智能速度偏移

解决了一个很实际的痛点：**市区限速经常只有 40-60，FSD 严格按限速走太慢，要反复手动调。**

我们加了两种偏移模式：
- **手动模式**：6 档可选（0% / 10% / 20% / 30% / 40% / 50%），一键切换
- **智能模式**：设定规则，比如"限速 60 以下偏移 30%，60-100 偏移 20%，100 以上偏移 10%"，系统根据当前路段限速自动应用

直接在车机上操作，开车时随时调整。

### 3. 网络隔离一键部署

路由器侧的 WiFi 隔离方案，从手动配置升级为 **一键安装 + Web 控制台**：
- `sh install.sh` 一条命令搞定全部配置
- 浏览器打开控制台就能管理黑白名单、查看 DNS 日志
- 自动检测 Tesla 新增的域名，动态更新规则

### 4. 全新 Web 控制页面

完全重构了 CAN mod 的 Web 界面，专门为车机屏幕优化：
- 仪表盘实时显示：油门、限速、跟车距离、方向盘握力等 CAN 信号
- 5 档驾驶风格切换（激进 → 适中 → 默认 → 保守 → 佛系）
- 功能开关一览无余：FSD、紧急车辆检测、ISA 抑制、Nag Killer 等

## 整体架构

```
┌─────────────────┐     CAN 总线      ┌──────────────────┐
│   Tesla 车机     │◄════════════════►│  CAN 模块 (ESP32) │
│                  │                   │  IP: 2.2.2.2      │
│  浏览器访问:     │     WiFi          │  Web 控制台: 80   │
│  http://2.2.2.2  │◄────────────────►│                    │
└────────┬─────────┘                   └────────┬───────────┘
         │                                      │
         │ WiFi (同一个路由器)                     │ WiFi STA
         │                                      │
    ┌────▼──────────────────────────────────────▼────┐
    │              OpenWrt 路由器                      │
    │                                                 │
    │  功能:                                          │
    │  - 提供 WiFi 接入点                              │
    │  - DNS 黑洞: *.tesla.cn → 127.0.0.1            │
    │  - 防火墙: 只放行导航等必需流量                    │
    │  - Web 控制台: http://路由器IP:8888              │
    │                                                 │
    │  IP: 192.168.x.x (管理) / 2.2.2.1 (车机侧)     │
    └─────────────────────────────────────────────────┘
```

**一句话原理：** Tesla 禁用 FSD 需要通过网络下发指令到车机。我们用路由器把这条通道掐断——车机连我们的 WiFi、关掉 4G，Tesla 服务器的命令就送不到。同时 CAN 模块直接修改总线信号，让 FSD 生效。

## 第一部分：CAN 模块

### 硬件

本方案使用 **微雪 ESP32-S3-RS485-CAN** 工业隔离模块：
- 芯片：ESP32-S3R8（240MHz，16MB Flash，8MB PSRAM）
- CAN 控制器：内置 TWAI，TX=GPIO15，RX=GPIO16
- 供电：USB-C 或 DC 5-36V
- 接口：RS485 + CAN + WiFi

你也可以用其他支持的硬件，见 `platformio.ini` 中的环境列表。

### CAN 总线原理（简单版）

Tesla 的车机和各控制模块之间通过 CAN 总线通信。FSD 是否启用、速度档位、跟车距离等设置，都是以 CAN 报文的形式在总线上传输的。

我们的模块做的事情：
1. **监听** 总线上的 CAN 报文（比如跟车距离设置）
2. **修改** 特定报文的特定字段（比如把 FSD 标志位设为 1）
3. **注入** 修改后的报文回到总线

车机收到修改后的报文，就会按新的配置运行。

### 首次刷写（USB）

**你需要准备：**
- USB-C 数据线（不是充电线！）
- 安装好 PlatformIO 的电脑（Mac / Linux / Windows）

**步骤：**

1. 先配置你的 WiFi 信息。编辑 `platformio.ini`，找到 `waveshare_esp32s3_rs485_can` 环境，修改：

```ini
'-DWIFI_STA_SSID="你的WiFi名"'
'-DWIFI_STA_PASS="你的WiFi密码"'
```

2. USB-C 连接模块到电脑

3. 运行烧录脚本：

```sh
sh flash_esp32.sh
```

脚本会自动：检测串口 → 编译 → 烧录 → 等待 WiFi 上线 → 验证状态。

也可以手动指定参数：

```sh
# 指定串口
sh flash_esp32.sh --port /dev/cu.usbmodem11201

# 指定 PlatformIO venv 路径
sh flash_esp32.sh --venv /path/to/your/venv

# 跳过烧录后验证
sh flash_esp32.sh --no-verify
```

4. 看到 `全部 OK` 就说明刷写成功。

### Web 控制台

模块启动后会同时开启：
- **AP 热点**：SSID `TeslaCAN`，密码 `canmod12`，访问 `http://192.168.4.1`
- **STA 模式**（如果配了 WiFi）：通过路由器网络访问 `http://2.2.2.2`

**仪表盘（Dashboard）：**
- 6 格实时数据：油门深度、融合限速、地图限速、车辆限速、方向盘握力、跟车距离
- 驾驶风格 5 档切换：Max → Hurry → Normal → Chill → Sloth
- 速度偏移控制：手动 6 档 / 智能规则

**设置（Settings）：**
- 硬件版本切换（LEGACY / HW3 / HW4）— 运行时切换，无需重刷
- 功能开关：FSD 绕过、ISA 抑制、紧急车辆检测、Nag Killer 等
- 每个开关都可以在车机上实时切换

**系统（System）：**
- WiFi SSID / 密码修改
- OTA 固件升级（选择 .bin 文件上传即可）
- 实时日志查看

### 装车

将模块通过 CAN 总线线缆连接到车辆的 X179 接口（后备箱驾驶员侧），接线方式参见 `guides/WIRING_GUIDE.md`。装好后通过 USB 或 DC 供电。

## 第二部分：网络隔离

### 为什么需要隔离？

Tesla 后台会定期检查车辆的配置状态。如果发现 FSD 被非正常启用，可以通过网络下发指令远程撤销。这个通道主要走 `hermes` 长连接协议，域名是 `*.tesla.cn`。

我们的策略：**让车机连我们控制的 WiFi，同时关掉 4G，切断所有上报和下发通道。**

### 隔离原理（两层防御）

**第一层 — DNS 黑洞：**

路由器上的 dnsmasq 把所有 Tesla 域名解析到 `127.0.0.1`：

```
车机查询 "hermes-prd.vn.cloud.tesla.cn"
    ↓
dnsmasq 返回 127.0.0.1
    ↓
车机尝试连接 127.0.0.1:443 → 失败
    ↓
Tesla 指令送不到
```

**第二层 — 防火墙：**

即使车机有 DNS 缓存或硬编码 IP，iptables 也会拦截：

```
iptables DNSONLY 链:
  ✅ 放行: DNS (53) / DHCP (67-68) / NTP (123)
  ✅ 放行: 白名单 IP (导航、connman 等必需服务)
  ❌ 拒绝: 其他所有出站流量
  ❌ 拒绝: 所有 IPv6 流量
```

**哪些功能保留？哪些被屏蔽？**

| 功能 | 状态 | 说明 |
|------|------|------|
| 导航 | 正常 | `nav-prd-maps.tesla.cn` 已放行 |
| FSD 驾驶 | 正常 | 不依赖网络 |
| WiFi 连接验证 | 正常 | `connman` 已放行 |
| 百度/QQ 等第三方 | 正常 | 非 Tesla 域名不受影响 |
| Tesla 手机 App 控车 | 不可用 | 走 hermes 通道，必须屏蔽 |
| 语音助手 | 不可用 | 走 Tesla API |
| OTA 升级 | 不可用 | 这正是我们要阻止的 |
| 车机 4G 数据 | 需手动关闭 | 否则会绕过路由器 |

### 路由器要求

- **系统**：OpenWrt（或类似的可刷 Linux 路由器）
- **必需命令**：dnsmasq、iptables、ip6tables、crond、tar
- **推荐型号**：TR341 或任何支持 OpenWrt 的路由器
- **说明**：不需要很强的性能，主要是跑 DNS 和防火墙规则

### 部署步骤

**1. 把安装包传到路由器：**

```sh
# 方法一：SCP（需要 SSH）
scp router-config/tesla-iso/dist/tesla-iso-1.0.3.tar.gz root@路由器IP:/tmp/

# 方法二：Python HTTP 服务器
cd router-config/tesla-iso/dist/
python3 -m http.server 8080
# 然后在路由器上: wget http://你的电脑IP:8080/tesla-iso-1.0.3.tar.gz -O /tmp/tesla-iso-1.0.3.tar.gz
```

**2. SSH 登录路由器执行安装：**

```sh
cd /tmp
tar -xzf tesla-iso-1.0.3.tar.gz
cd tesla-iso-1.0.3
sh install.sh
```

脚本会自动：
- 检查依赖
- 备份旧配置
- 部署 DNS 黑名单 + 防火墙规则 + 定时任务 + Web 控制台
- 添加 `2.2.2.1/24` 网段（给车机访问 CAN 模块用）
- 跑 13 项自检

**3. 也可以用一键部署脚本（从你的电脑远程操作路由器）：**

```sh
sh router-config/tesla-iso/deploy.sh \
   --router-ip 192.168.2.254 \
   --user admin --pass 你的路由器密码 \
   --wifi-ssid 你的WiFi名 --wifi-pass 你的WiFi密码
```

### 路由器 Web 控制台

安装完成后，在电脑或手机浏览器打开 `http://路由器IP:8888`：

- **系统状态**：dnsmasq / crond / 连接设备数 / 例外数
- **例外管理**：添加 / 删除放行的域名
- **待审页**：自动列出车机查询过但被屏蔽的 Tesla 子域名——你可以根据需要选择放行
- **DNS 日志**：最近的 DNS 查询记录
- **升级**：上传新版安装包，一键升级

> 注意：控制台只允许内网 IP（192.168.x.x 等）访问，车机浏览器打不开，安全无忧。

### 关于 "假公网" IP（2.2.2.0/24）

你可能会好奇为什么不用 `192.168.x.x`。原因是 **Tesla 车机浏览器会拒绝访问 RFC1918 私有地址**（192.168.x.x、10.x.x.x 等）。

我们的方案：在路由器上添加一个 `2.2.2.1/24` 的 secondary IP，ESP32 模块用 `2.2.2.2`。这个地址段虽然是公网地址，但在我们的局域网内是完全隔离的——路由器不会把 `2.2.2.0/24` 的流量转发到真正的公网。

车机浏览器看到的是"公网 IP"，可以正常打开 `http://2.2.2.2`。

## 第三部分：车机配置

装好模块、部署好路由器后，在车上操作：

1. **连接 WiFi**：车机设置 → WiFi → 连接你部署时设置的 SSID
2. **关闭 4G 数据**：车机设置 → 网络 → 关闭蜂窝数据
3. **重启车机**：按住方向盘左右两个滚轮 10 秒
   - **这一步必须做！** 清除车机里的旧 DNS 缓存和已建立的 hermes 长连接
4. **打开浏览器**：访问 `http://2.2.2.2`，看到 CAN Mod 控制台就说明一切就绪

## 常见问题

### Q: 刷写失败怎么办？

- 确认 USB-C 是**数据线**，不是充电线
- 关闭所有占用串口的程序（Serial Monitor、其他终端）
- 如果 Mac 看不到 `/dev/cu.usbmodem*`，按一下模块上的 RST 按键
- 试试手动指定串口：`sh flash_esp32.sh --port /dev/cu.usbmodemXXXX`

### Q: WiFi 连不上怎么办？

- 检查 `platformio.ini` 里的 SSID 和密码是否正确
- 确认路由器的 2.4GHz WiFi 已开启（ESP32 不支持 5GHz）
- 模块的 AP 热点 `TeslaCAN` 始终可用，连上后访问 `http://192.168.4.1` 修改 WiFi 配置

### Q: 车机浏览器打不开 2.2.2.2？

- 确认路由器已添加 `2.2.2.1/24` secondary IP：SSH 执行 `ip addr show br-lan`
- 确认车机已连上路由器 WiFi
- 确认车机 4G 已关闭
- 尝试在车机浏览器地址栏输入完整地址：`http://2.2.2.2/`

### Q: 导航不能用了？

- 检查 `nav-prd-maps.tesla.cn` 是否在白名单中：打开路由器控制台 `http://路由器IP:8888`，查看例外列表
- 如果不在，手动添加例外
- 执行 `sh verify.sh` 看哪些域名有问题

### Q: 还能用手机 App 控车吗？

不能。手机 App 控车走的是 hermes 通道（`hermes-*.tesla.cn`），和 Tesla 下发禁用指令用的是同一条路。放行 hermes = 放弃防护，所以必须屏蔽。

### Q: 路由器重启后配置会丢失吗？

不会。所有配置都写在 `/etc/` 持久化分区，防火墙规则在开机时自动重建。

### Q: 如何升级固件？

两种方式：
1. **USB 刷写**：电脑连模块，运行 `sh flash_esp32.sh`
2. **OTA 无线升级**：在 Web 控制台的 System 标签页上传 `.bin` 文件
